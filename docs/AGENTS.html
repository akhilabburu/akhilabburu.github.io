Commands and Development Workflow

The primary entry point for applying and deploying updates is the provided Windows batch script:

  • Full Build & Deploy: Execute update_site.bat in the project root. This orchestrates the entire build pipeline:
    1. Generates the monthly link .qmd files via R.
    2. Executes quarto render to compile the static site into the docs/ folder.
    3. Automatically checks for differences, stages the docs/ folder, and makes a git commit (with a generated timestamp message or user prompt) and pushes it to GitHub.

Manual Commands

If isolating certain tasks is necessary: - Build Links Subdirectory: Execute Rscript scripts/generate_monthly_links.R. (Note: update_site.bat uses a statically configured local path to Rscript.exe; adjust if the environment differs). - Compile Site: Run quarto render directly to build the site locally. - Dev Preview: Run quarto preview to start a local hot-reloading development server.

Shell Pitfalls (PowerShell)

  • Rscript is not on PATH. Use the call operator with the full path: & 'C:\Program Files\R\R-4.4.1\bin\Rscript.exe' scripts/generate_monthly_links.R. Do not use cmd /c or double-quoted paths — they fail with PowerShell’s parser.
  • Quarto preview can leave stale session files in .quarto/. If quarto preview throws a NotFound / stat error, delete the .quarto directory and retry: Remove-Item -Recurse -Force .quarto; quarto preview.
  • Quarto preview useful flags: --port 4321 --no-browser for headless verification.

High-Level Architecture

This repository operates as a Quarto-based static website (_quarto.yml) utilized as a personal/academic portfolio, statically served via GitHub Pages directly from the docs/ directory.

  • Automated Links Subsystem: The links_database.csv file is the single source of truth for curated links. The R script scripts/generate_monthly_links.R reads this CSV and generates:

    1. Monthly .qmd files (e.g., links/2025-04.qmd) — one per month, with human-readable titles (“April 2025”), colored category dots, and source domains.
    2. links/index.qmd — a single-page browsing experience with all links inline, interactive JS tag filtering (toggle by category), and clickable month headers linking to the monthly pages.

    Any modifications to links should be directed entirely to links_database.csv. Never hand-edit the generated links/*.qmd files — they are overwritten on each run. The force_update = TRUE flag in the R script always regenerates all files.

    CSS for the links section lives in assets/styles.css (tag filter bar, colored dots/pills, link items, month groups). Category colors use the .cat-{name} class convention shared between the filter bar tags and the link dot indicators.

  • Project Map:

    • docs/: The designated build output directory. Do not manually edit files here, as quarto render overwrites changes.
    • links/: Generated monthly pages and the main index. Do not hand-edit — generated by the R script.
    • iitd/: A structured directory holding specific guides, tools, and scripts tailored to navigating the Indian Institute of Technology Delhi (e.g., proxy authentication, booklet generation, campus WiFi).
    • projects/: Standalone pages dedicated to interactive widgets or specific topics (cricket stats, maps, etc.).
    • index.qmd & root documents: Control the initial entry points and primary navigation nodes. The output.qmd functions as the persistent location for academic presentation and publication manifests.
    • assets/: Stores shared CSS styling (styles.css) and custom light/dark theme SCSS overrides (darkly.scss, flatly.scss) referenced by the main Quarto configuration.
